Dynamo Dancer

Dynamo Dancer is an augmented reality dancing mobile game I made with with a team of 4 other students. The game enables the player to use their own body and webcam to play the game.

As the main developer in this team I was tasked with implementing our ideas into Unity. Such ideas include the motion capture technology I developed for this game and a fully working level editor that also works on mobile.

Skrypt

Skrypt is a general purpose scripting language. It's an interpreted language thats compiled at runtime, is weakly typed supports object-oriented, functional and imperative programming styles. The goal of Skrypt is to create an easy to use scripting language that combines familiar ideas and syntax from other languages. Skrypt includes has an extensive standard library and extensive interoperability with .NET.

This project is open source, learn more here.

Fright Night

Fright Night is a 2D, top-down horror game in which you must find and escape from a school with as many of your friends as possible, while trying to survive the presence of a killer clown. The player can make use of various items to fend off the clown.

With this project I learned a lot about designing game AI which is used for the killer clown and in-game friends.

Bubba Bird

After an exhausting day in a typical bird’s life, Bubba the bird falls asleep in a tree. Unfortunately, Bubba loses grip on the branch he’s sleeping on, and he falls on top of a bee’s nest hanging lower in the tree. The bees get angry at Bubba and try to attack him. Bubba must now flee from all the bees. As Bubba flees, he encounters various obstacles, such as bees, giant rainworms, and more bee’s nests.

Seat of Burden

Seat of Burden was a collaborative game-project that aimed to teach the player about cultural awareness. It's set in an imaginary medieval realm; Culturia. The premise of the game is that you're the son of a king who died. The king hasn't ruled Culturia in the way that suited it's inhabitants. As the new king of Culturia, you aim to restore the realm to it's former glory.

I was tasked with most of the development work. Because I was only using a single library to render items dynamically, I didn't have robust base to work on such as an actual game engine. This meant I basically had to write a card game engine from scatch.

Python Brain**** Interpreter

I got pretty interested in the esoteric programming language 'Brainf***'. It's a language that's meant to be Turing-complete while only using 8 different characters for a program. I wanted to learn a bit of Python (another programming language), so I decided to make my own Brainf*** interpreter in Python.
